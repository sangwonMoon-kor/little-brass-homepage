# 리틀브라스 편집형 리디자인 명세

## 배경

현재 사이트는 기능과 기본 접근성은 안정적이지만, 필기체 영문 레이블·금색 포인트·둥근 카드·원형 아이콘·어두운 사진 오버레이가 페이지마다 반복된다. 이 조합은 실제 학원의 개성보다 범용적인 “프리미엄 AI 랜딩페이지” 인상을 먼저 만든다. 홈의 학원 사진도 화면을 크게 점유해 정보 위계가 약해지고 스크롤이 길게 느껴진다.

이번 리디자인은 기존 Hono/JSX 구조, 콘텐츠, URL, 예약 연결과 실제 사진 자산을 유지하면서 시각 언어와 정보 구조를 정돈한다.

## 목표

- 리틀브라스를 실제 운영되는 금관악기 전문 학원답게 보이게 한다.
- 한국어 정보와 실제 교육 공간을 중심에 두고 장식적인 영문 문구를 줄인다.
- 홈의 학원 사진 크기와 반복을 줄여 내용이 더 빨리 읽히게 한다.
- 모든 페이지에 하나의 편집형 디자인 시스템을 적용한다.
- Emil Kowalski의 원칙에 맞춰 상호작용은 빠르고 절제되며 자연스럽게 유지한다.

## 비목표

- 새로운 CMS, 예약 시스템 또는 프레임워크를 도입하지 않는다.
- 사실 확인이 필요한 수업 내용, 가격, 합격 실적을 임의로 새로 만들지 않는다.
- 보유 사진을 AI 이미지로 교체하지 않는다.
- `littlebrass.com` 연결 작업은 이번 시각 리디자인 범위에 포함하지 않는다.

## 디자인 방향: 편집형 음악 아카데미

음악회 프로그램북과 독립 교육기관의 인쇄물을 참고한 차분한 편집형 화면으로 전환한다. 장식으로 “고급스러움”을 설명하지 않고, 타이포그래피·정렬·실제 사진·구체적인 정보가 전문성을 증명하게 한다.

### 시각 원칙

1. **한국어 우선**: 핵심 제목과 설명은 한국어로 쓴다. 영문은 `Little Brass` 브랜드명이나 악기명처럼 의미가 있을 때만 보조적으로 사용한다.
2. **금관색은 포인트**: 금색 면, 금색 아이콘, 금색 테두리를 반복하지 않는다. 링크 상태, 작은 구분선, 번호 등 제한된 요소에만 탁한 브라스 색을 사용한다.
3. **카드보다 편집 구조**: 동일한 둥근 카드 묶음을 줄이고 선, 열, 여백, 번호로 정보를 구분한다.
4. **사진은 증거**: 사진을 배경 장식이 아니라 공간과 교육을 보여주는 자료로 사용한다. 과도한 오버레이와 이미지 위 문구를 줄인다.
5. **비대칭 위계**: 모든 섹션을 중앙 정렬하지 않는다. 제목 열과 본문 열을 분리하고, 사진 크기를 콘텐츠 중요도에 따라 다르게 한다.

## 디자인 토큰

### 색상

- Paper: `#F4F0E8` — 사이트 기본 배경
- Surface: `#FCFAF6` — 필요한 경우에만 사용하는 밝은 면
- Ink: `#20221F` — 제목과 주요 텍스트
- Muted Ink: `#62645E` — 보조 설명
- Brass: `#9A7428` — 제한된 강조
- Rule: `#D8D1C4` — 구분선
- Deep Ink: `#171916` — 푸터와 필요한 어두운 면

브라스 색은 전체 화면 면적의 약 5% 이하로 제한한다. 그라디언트와 장식용 그림자는 사용하지 않는다.

### 타이포그래피

- 한국어 제목: `Noto Serif KR`, 보통 또는 세미볼드
- 본문과 UI: `Noto Sans KR`, 보통 또는 미디엄
- 영문 브랜드: `Cormorant Garamond`, 로고와 소수의 브랜드 표기에만 사용
- `Dancing Script`와 반복적인 이탤릭 `Playfair Display` 사용은 제거한다.
- 제목은 불필요한 영문 소제목 없이 문장형 한국어로 작성한다.
- 본문 한 줄 길이는 데스크톱 기준 약 55~68자로 제한한다.

### 형태와 간격

- 기본 콘텐츠 최대 너비: `1180px`
- 본문 콘텐츠 최대 너비: `720px`
- 섹션 세로 간격: 데스크톱 `96~128px`, 모바일 `64~80px`
- 기본 모서리: 이미지 `0~4px`, 기능적 컨트롤 `6~10px`
- 필 형태 버튼, 원형 아이콘 배경, 일괄적인 16px 카드 라운드는 제거한다.

## 전역 구성

### 내비게이션

- 상단 정보 바를 제거하고 연락처는 푸터와 위치 페이지에서 제공한다.
- 로고, 주요 메뉴, `원데이 클래스` 단일 행동만 남긴다.
- 홈 첫 화면에서는 투명 또는 반투명 내비게이션을 사용하고, 스크롤 후 Paper 배경과 하단 구분선으로 전환한다.
- 현재 페이지는 색상보다 얇은 밑줄과 `aria-current`로 표시한다.

### 버튼과 링크

- 핵심 CTA는 각 영역에 하나만 둔다.
- 기본 CTA는 직사각형에 가까운 단색 버튼으로 사용한다.
- 보조 행동은 테두리 버튼 대신 텍스트 링크와 화살표로 표현한다.
- 눌림 상태는 `scale(0.97)`, UI 전환은 300ms 이하로 유지한다.

### 푸터

- 3열 링크 묶음과 아이콘 나열을 줄이고 브랜드, 주소·연락처, 예약·소셜 링크를 한 줄 위계로 재구성한다.
- 어두운 배경은 푸터 한 곳에서만 사용해 페이지의 마침표 역할을 하게 한다.

## 홈 화면

### 1. 히어로

- 영상은 리틀브라스의 실제 분위기를 전달하므로 유지한다.
- 높이는 `clamp(620px, 82vh, 820px)`로 제한해 항상 한 화면보다 과도하게 크지 않게 한다.
- 중앙 정렬된 필기체 영문 문구와 세로 장식선을 제거한다.
- 콘텐츠는 왼쪽 아래에 정렬한다.
- 제목은 `금관악기를 처음 만나는 곳, 리틀브라스`처럼 구체적인 한국어로 구성한다.
- 행동은 `원데이 클래스 예약` 하나를 주 버튼으로 두고 `커리큘럼 보기`는 텍스트 링크로 처리한다.

### 2. 교육 대상과 악기

- `What We Offer` 필기체 레이블을 제거한다.
- 핵심 문장과 트럼펫·호른·트롬본·유포늄 악기 목록을 하나의 편집형 인덱스로 합친다.
- 네 개의 동일한 이미지 카드는 사용하지 않는다. 대표 악기 사진 한 장과 악기명 목록을 5:7 비율의 비대칭 열로 배치한다.

### 3. 리틀브라스를 선택하는 이유

- 현재의 화면 폭 절반을 차지하는 3개 지그재그 사진 행을 압축한다.
- 각 항목은 번호, 제목, 설명, 중간 크기 사진으로 구성한 편집형 행으로 바꾼다.
- 데스크톱에서 사진 열은 콘텐츠 폭의 약 38~42%, 텍스트 열은 58~62%를 사용한다.
- 사진은 `4:3` 또는 `5:4`, 최대 높이 `420px`로 제한한다.
- 모바일 사진은 폭 100%, 최대 높이 `320px`로 제한하고 텍스트 다음에 배치한다.
- 세 항목 모두 동일한 큰 이미지로 반복하지 않고, 두 항목에만 사진을 사용하거나 마지막 항목은 텍스트·운영 정보로 마무리한다.

### 4. 원데이 클래스

- 별도의 둥근 카드 밴드 대신 Paper와 한 줄 구분선 안에서 제목, 짧은 안내, 예약 행동을 배치한다.
- 전화 상담은 보조 텍스트 링크로 유지한다.

### 5. 학원 공간 사진

- 실제 학원 사진은 유지하되 홈페이지에서 차지하는 총 높이를 줄인다.
- 섹션 전체는 최대 너비 `1180px` 안에 둔다.
- 대표 사진은 `16:10`, 최대 높이 `500px`로 제한한다.
- 보조 사진 두 장은 `4:3`, 데스크톱 최대 높이 `280px`, 모바일 최대 높이 `240px`로 제한한다.
- 사진 위 영문 라벨과 검은 그라디언트 오버레이를 제거하고, 사진 아래 한국어 캡션을 둔다.
- 더 많은 공간 사진은 갤러리 페이지로 연결한다.

### 6. 소식

- 동일한 3열 블로그 카드를 날짜·제목·요약이 정렬된 세로 목록으로 전환한다.
- 그림자와 호버 상승 효과를 제거하고 행 구분선과 미세한 화살표 이동만 사용한다.

### 7. 마지막 행동

- 배경 사진 + 검은 오버레이 + 금색 영문 문구 조합을 제거한다.
- 짧은 상담 안내와 예약 버튼을 Paper 또는 Ink 단색 영역으로 마무리한다.

## 하위 페이지

### 공통 헤더

- 모든 페이지의 300px 사진 배너와 중앙 정렬 영문 필기체를 제거한다.
- Paper 배경 위에 페이지 제목, 한 문장 설명, 작은 페이지 번호 또는 구분선을 배치한다.
- 필요한 경우 오른쪽에 작은 실제 사진을 추가하되 높이는 `220~280px`로 제한한다.

### 교육철학

- 원장 소개와 교육 철학을 긴 읽기 흐름으로 재구성한다.
- 2×2 아이콘 카드는 번호가 있는 텍스트 목록으로 바꾼다.
- 검은 전체 폭 브랜드 스토리 영역은 Paper 계열 면과 인용형 타이포그래피로 전환한다.

### 커리큘럼

- 반복되는 16px 라운드 단계 카드를 줄이고 악기별 탭 또는 목차형 앵커와 단계별 표/목록으로 정리한다.
- 반복되는 원형 아이콘과 `성취 포인트` 배지를 제거한다.
- 기존 수업 정보는 유지하되 중복 문구를 정리하고 비교가 쉬운 열 구조를 사용한다.

### 갤러리

- 정사각형 3열 격자를 잡지형 비대칭 그리드로 바꾼다.
- 사진 크기는 한 장만 크게, 나머지는 중간·작은 크기로 구성한다.
- 오버레이 대신 캡션을 사용한다.

### 찾아오시는 길

- 지도 링크를 큰 빈 카드 대신 주소와 `네이버 지도에서 보기` 행동이 결합된 정보 블록으로 만든다.
- 교통수단 3개 카드는 구분선이 있는 목록으로 전환한다.
- 실제 운영시간과 연락처를 가장 빠르게 찾을 수 있도록 첫 콘텐츠 영역에 배치한다.

## 모션과 상호작용

- 모션은 의미가 있을 때만 사용한다.
- 버튼·링크는 140~220ms, 메뉴와 이미지 상태 변화는 최대 280ms로 제한한다.
- 기본 이징은 `cubic-bezier(0.22, 1, 0.36, 1)` 계열을 사용한다.
- `transform`과 `opacity` 중심으로 구현하고 `transition: all`은 사용하지 않는다.
- 호버 전용 효과는 `(hover: hover) and (pointer: fine)`에서만 적용한다.
- `prefers-reduced-motion`에서는 이동 거리와 이미지 확대를 제거하고 짧은 페이드만 남긴다.

## 접근성과 반응형

- 모든 텍스트와 링크는 WCAG AA 대비를 충족한다.
- 키보드 포커스는 브라스색 외곽선 또는 명확한 밑줄로 표시한다.
- 사진 캡션과 대체 텍스트는 장식이 아닌 실제 내용을 설명한다.
- 모바일에서 데스크톱의 비대칭 구조를 억지로 유지하지 않고 제목 → 설명 → 사진 → 행동 순으로 단순화한다.
- 터치 대상은 최소 44px를 유지한다.

## 검증 기준

- 홈과 모든 공개 페이지에서 Dancing Script가 보이지 않는다.
- 핵심 섹션에 동일한 3열/4열 둥근 카드 묶음이 반복되지 않는다.
- 홈의 실제 학원 사진은 명세한 최대 높이를 넘지 않는다.
- 데스크톱 1440px, 태블릿 768px, 모바일 390px에서 정보 순서와 사진 크기가 자연스럽다.
- 모든 공개 링크, 예약 연결, 전화, 메일, 네이버 지도 연결이 유지된다.
- 타입 검사, 빌드, 테스트가 통과하고 접근성/모션 회귀가 없다.
