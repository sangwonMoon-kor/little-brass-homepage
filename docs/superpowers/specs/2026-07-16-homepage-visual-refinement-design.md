# 리틀브라스 홈 화면 시각 개선 명세

## 상태와 범위

이 문서는 브라우저 시안으로 승인된 리틀브라스 홈 화면의 최종 디자인을 정의한다. `2026-07-15-editorial-redesign-design.md`의 공통 편집형 원칙은 유지하되, 홈 화면의 색상·섹션 순서·사진 사용·예약 행동에 관한 내용은 이 문서가 우선한다.

이번 작업은 홈 화면과 홈에서 보이는 공통 내비게이션·푸터 상태만 다룬다. 커리큘럼, 교육철학, 갤러리, 찾아오시는 길 페이지의 본문 디자인은 변경하지 않는다.

## 목표

- 사이트 전체의 기본 면을 순수한 흰색으로 정리한다.
- 네이비를 넓은 구분 면과 핵심 행동에만 사용해 리틀브라스의 정체성을 분명하게 만든다.
- 히어로 영상과 텍스트가 서로 방해하지 않도록 완전히 분리한다.
- 악기 이름과 설명의 중복을 줄이면서 네이비 악기 띠가 만드는 섹션 구분은 유지한다.
- 실제 학원 사진은 교육과 공간을 설명하는 곳에만 제한적으로 사용한다.
- 홈 화면의 원데이 클래스 예약 행동을 처음과 끝, 정확히 두 곳에만 배치한다.
- 데스크톱과 모바일 모두 불필요한 빈 공간이나 가로 넘침 없이 자연스럽게 읽히게 한다.

## 비목표

- 예약 시스템, 블로그 연동, 연락처 데이터 또는 공개 URL을 바꾸지 않는다.
- 실제 학원 사진을 바탕으로 승인된 보정·크롭본 외에 새로운 합성 장면을 추가하지 않는다.
- 하위 페이지의 정보 구조를 다시 설계하지 않는다.
- `littlebrass.com` 도메인 연결은 이번 작업에 포함하지 않는다.
- 새 UI 프레임워크, 애니메이션 라이브러리 또는 런타임 의존성을 추가하지 않는다.

## 디자인 시스템

### 색상

- Base White: `#FFFFFF` — 홈의 기본 배경
- Navy: `#102B4E` — 악기 인덱스, 최종 CTA, 주요 버튼
- Deep Navy: `#0A1C33` — 상단 정보 띠와 푸터
- Brass: `#B79035` — 작은 레이블, 번호, 최종 CTA 버튼
- Muted Brass: `#8B681E` — 흰 배경 위 작은 강조 텍스트
- Ink: `#17202C` — 일반 본문
- Navy Ink: `#0A1C33` — 큰 제목
- Muted: `#626A73` — 보조 설명
- Rule: `#DEDEDE` — 얇은 구분선
- Soft Surface: `#F8F8F6` — 최근 소식 카드에만 허용

그라디언트, 장식용 그림자, 둥근 카드, 넓은 금색 면은 사용하지 않는다. 사진 위 캡션이 필요하면 하단의 평평한 반투명 Navy 면으로 가독성을 확보한다.

### 타이포그래피

- 브랜드 워드마크: `Cormorant Garamond`
- 한국어 제목: `Noto Serif KR`
- 본문과 UI: `Noto Sans KR`

데스크톱의 핵심 제목은 `42~54px`, 섹션 제목은 `34~44px`, 항목 제목은 `20~24px` 범위로 사용한다. 모바일에서는 줄바꿈을 우선하고 제목을 `32~38px` 범위로 낮춘다. 제목은 공간을 채우기 위해 과도하게 키우지 않는다.

## 홈 정보 구조

홈은 다음 순서를 고정한다.

1. 데스크톱 정보 띠와 내비게이션
2. 실제 수업 영상
3. 영상 아래 히어로 소개와 첫 예약 행동
4. 네이비 악기 인덱스 띠
5. 네 가지 악기 사진과 설명
6. 리틀브라스 교육 기준
7. 실제 학원 공간
8. 최근 소식
9. 사진 없는 최종 예약 CTA
10. 푸터

중간 예약 띠는 제거한다. 섹션의 순서는 데스크톱과 모바일에서 동일하다.

## 섹션 명세

### 1. 상단 정보와 내비게이션

- 데스크톱에서만 높이 약 `30px`의 Deep Navy 정보 띠를 사용한다.
- 정보 띠에는 학원 주소와 상담 전화번호만 표시하고 예약 행동은 넣지 않는다.
- 주 내비게이션은 흰 배경, 얇은 하단 구분선, 최대 너비 `1180px` 구조를 사용한다.
- 홈에서는 데스크톱과 모바일 내비게이션의 원데이 클래스 버튼을 렌더링하지 않는다.
- 하위 페이지에서는 기존 내비게이션 예약 링크를 유지해 예약 접근성을 보존한다.
- 모바일에서는 정보 띠를 숨기고 브랜드와 메뉴 버튼만 표시한다.

### 2. 히어로 영상과 소개

- 영상은 전체 너비의 독립 영역으로 표시하고 그 위에 제목, 설명 또는 버튼을 올리지 않는다.
- 어두운 스크림을 제거한다. 영상 자체의 색과 장면이 그대로 보이게 한다.
- 영상 우측 하단에는 작은 `리틀브라스 실제 수업 영상` 배지만 허용한다.
- 영상 아래 흰 배경 소개 영역은 데스크톱에서 제목과 설명·행동을 두 열로 배치한다.
- 제목은 `금관악기의 첫 소리부터, 오래 남는 음악까지`를 사용한다.
- 설명 아래에 홈의 첫 번째 `원데이 클래스 예약` 버튼과 보조 `커리큘럼 보기` 링크를 둔다.
- 예약 링크가 외부 네이버 예약으로 열린다는 안내는 작은 보조 문구로 유지한다.

### 3. 네이비 악기 인덱스

- 히어로 소개 바로 아래 전체 너비 Navy 띠를 배치해 영상·소개 영역과 악기 콘텐츠를 구분한다.
- 첫 칸은 `네 가지 금관악기`와 짧은 안내 문구를 표시한다.
- 나머지 네 칸은 `01 트럼펫`, `02 호른`, `03 트롬본`, `04 유포늄`만 표시한다.
- 이 띠에는 음색 설명을 반복하지 않는다. 음색 설명은 바로 아래 사진 카드가 담당한다.
- 데스크톱은 5열, 태블릿과 모바일은 2열로 접는다. 첫 안내 칸은 모바일에서 전체 너비를 차지한다.

### 4. 악기 사진과 설명

- 흰 배경 위에 제목·소개와 네 악기 사진을 배치한다.
- 데스크톱에서는 네 장을 동일 높이의 한 줄로 표시한다.
- 각 사진 하단에는 한국어 악기명, 한 줄 음색 설명, 작은 영문 악기명을 함께 둔다.
- 사진 하단 정보는 그라디언트 대신 일정한 높이의 반투명 Navy 면을 사용한다.
- 문구는 다음을 고정한다.
  - 트럼펫 — 선명하고 곧은 소리
  - 호른 — 깊고 따뜻한 울림
  - 트롬본 — 넓고 힘 있는 음색
  - 유포늄 — 부드럽고 풍성한 소리
- 모바일에서는 한 열 또는 화면 폭이 허용하는 두 열로 전환하되 글자가 사진 밖으로 잘리지 않아야 한다.

### 5. 교육 기준

- 전체 배경은 흰색을 유지한다.
- 섹션의 위와 아래는 최근 소식 섹션과 같은 `1px` Rule 색상의 회색 선으로 구분한다. 두꺼운 Navy 띠는 사용하지 않는다.
- 왼쪽에는 실제 학원 사진을 보정한 `brand-wall-01.webp` 한 장을 사용하고 오른쪽에는 제목·소개·세 가지 기준을 배치한다.
- 승인된 사진은 원본 `display-04.webp`에서 민트색 수납장, 천장, 바닥과 오른쪽 돌출 벽을 제외하고 중앙의 로고·곡선 진열창·하단 벽 패널만 남긴 세로 구도다.
- 파일 자체가 최종 구도이므로 CSS 확대나 위치 이동으로 다시 크롭하지 않는다. `object-fit: cover`와 중앙 정렬만 사용한다.
- 로고의 `LITTLE BRASS` 글자와 진열 악기, 벽 패널의 수직선은 보정본 그대로 유지한다.
- 사진 자체에 별도 Navy 세로선, 테두리 또는 장식 그림자를 추가하지 않는다.
- 세 가지 기준은 얇은 회색 선으로만 구분하고 번호는 Muted Brass, 제목은 Navy로 표시한다.
- 제목과 항목 글자는 기존 실제 화면보다 한 단계 크게 하되 오른쪽 열 안에서 두 줄 이상 과도하게 깨지지 않게 한다.

### 6. 학원 공간

- 순수 흰색 배경을 유지한다.
- 데스크톱 사진 세 장은 한 줄 `2:1:1` 비율로 배치하고 모두 `420px`의 같은 높이를 사용한다.
- 왼쪽에는 대기 공간, 오른쪽 두 칸에는 개인 연습실과 레슨실을 표시한다.
- 사진 캡션은 작고 명확하게 유지한다.
- 모바일에서는 대표 사진 한 장 아래에 보조 사진 두 장을 두 열로 배치한다.
- 그리드 트랙은 `minmax(0, 1fr)`를 사용해 가로 넘침과 좌측 하단의 큰 빈 공간을 방지한다.

### 7. 최근 소식

- 흰 배경과 Soft Surface 카드의 대비를 사용한다.
- 데스크톱은 3열, 모바일은 1열로 표시한다.
- 카드는 회색 외곽선과 상단 `3px` Brass 선만 사용한다.
- 날짜, 제목, 짧은 설명을 보여주며 현재 블로그 데이터와 빈 상태 처리를 그대로 사용한다.
- 그림자, 큰 라운드, 과도한 상승 호버는 사용하지 않는다.

### 8. 최종 CTA

- 최종 영역에는 사진을 사용하지 않는다.
- 전체 Navy 배경 위 최대 너비 `1180px`의 텍스트 레이아웃을 둔다.
- 데스크톱은 약 `6:4` 비율의 두 열로 구성한다.
  - 왼쪽: 작은 Brass 레이블과 `어떤 악기로 시작할지 함께 찾아드립니다` 제목
  - 오른쪽: 짧은 설명과 Brass 배경의 두 번째 `원데이 클래스 예약` 버튼
- 높이는 약 `352px`로 제한해 푸터 앞에 지나치게 큰 공백을 만들지 않는다.
- 모바일에서는 제목, 설명, 버튼 순서의 한 열로 전환한다.

### 9. 푸터

- Deep Navy 배경과 흰색·반투명 흰색 텍스트를 사용한다.
- 브랜드, 주소·연락처, 인스타그램·유튜브 링크를 표시한다.
- 홈에서는 `네이버 예약` 푸터 링크를 렌더링하지 않는다.
- 하위 페이지에서는 기존 예약 접근을 유지할 수 있지만 한 화면에 복수의 강조 예약 버튼을 만들지 않는다.

## 예약 행동 계약

홈 HTML에서 `SITE.reservationUrl`을 사용하는 사용자 노출 링크는 정확히 두 개여야 한다.

1. 히어로 소개의 첫 예약 버튼
2. 사진 없는 최종 CTA의 마지막 예약 버튼

홈에서는 다음 예약 노출을 제거한다.

- 데스크톱 내비게이션 예약 버튼
- 모바일 메뉴 예약 버튼
- 교육 기준 아래 중간 예약 띠
- 푸터의 네이버 예약 링크

두 예약 링크는 모두 새 창으로 열리고 `rel="noopener noreferrer"`를 유지한다. 전화 상담과 커리큘럼은 예약 버튼과 다른 위계의 텍스트 링크로 제공한다.

## 구조와 파일 경계

기존 Hono JSX와 정적 자산 파이프라인을 유지한다.

- `src/pages/HomePage.tsx`: 홈 섹션 순서, 악기 데이터, 교육 기준, 공간, 소식, 두 예약 행동을 담당한다.
- `src/components/Navigation.tsx`: `pathname === '/'`일 때 예약 링크를 생략하고 하위 페이지에서는 기존 행동을 유지한다.
- `src/components/Layout.tsx`: 홈 푸터에서 예약 링크를 생략하고 데스크톱 정보 띠를 제공한다.
- `public/static/style.css`: 흰색·Navy·Brass 토큰, 홈 레이아웃, 반응형, 상호작용을 담당한다.
- `tests/accessibility-structure.test.ts`: 홈 예약 링크 두 개, 단일 `h1`, 내비게이션·푸터 예약 중복 제거를 검증한다.
- `tests/assets.test.ts`: 히어로 영상, WebP 사진, 이미지 크기 힌트와 공간 그리드 수축을 검증한다.
- `tests/build-output.test.ts`: 그라디언트·장식 효과·과도한 모션 회귀와 빌드 결과를 검증한다.

별도 상태 관리나 새 컴포넌트 계층은 추가하지 않는다. 블로그 글은 기존 `posts` prop을 그대로 사용하고, 사이트 정보와 외부 링크는 `SITE` 설정을 단일 출처로 유지한다.

## 데이터·오류 처리

- 블로그 로딩 실패 시 현재의 빈 상태 메시지를 유지한다.
- 영상 로딩 실패 시 기존 WebP 포스터가 보이게 한다.
- 하단 이미지는 `loading="lazy"`, `decoding="async"`, 명시적 `width`·`height`를 유지한다.
- 외부 링크에는 기존 보안 속성을 유지한다.
- JavaScript가 비활성화되어도 모든 핵심 텍스트, 사진과 링크는 읽을 수 있어야 한다.

## 모션과 상호작용

- 버튼 누름은 `transform: scale(0.97)`과 `140~160ms` ease-out을 사용한다.
- 링크 색상과 밑줄 변화는 `160~220ms` 범위로 제한한다.
- 사진 호버 확대는 fine pointer 환경에서만 최대 `scale(1.025)`를 사용한다.
- `transition: all`, `ease-in`, 장시간 자동 모션을 사용하지 않는다.
- `prefers-reduced-motion`에서는 이동과 사진 확대를 제거한다.

## 접근성과 반응형

- 홈에는 정확히 하나의 `h1`이 있어야 한다.
- 모든 텍스트와 버튼은 WCAG AA 대비를 충족한다.
- 포커스 표시와 최소 `44px` 터치 영역을 유지한다.
- 영상은 장식 콘텐츠로 처리하되 실제 수업 영상임을 보조 배지로 알린다.
- 사진의 대체 텍스트는 악기 또는 실제 학원 공간을 구체적으로 설명한다.
- `1440px`, `768px`, `390px`에서 가로 넘침이 없어야 한다.
- 모바일 정보 순서는 영상 → 소개 → 악기 띠 → 악기 사진 → 교육 기준 → 공간 → 소식 → 최종 CTA로 유지한다.

## 검증 기준

- 홈의 기본 배경이 `#FFFFFF`이고 따뜻한 베이지 또는 연두색 대면이 남지 않는다.
- 히어로 영상 위에 제목이나 예약 버튼이 겹치지 않는다.
- 히어로 아래 네이비 악기 인덱스가 보이고 음색 설명은 사진 카드에만 나타난다.
- 교육 기준 섹션의 상·하단에 동일한 `1px` 회색 선이 있고, 승인된 세로 사진에 수납장·천장·바닥·오른쪽 돌출 벽이 보이지 않는다.
- 학원 공간 사진 세 장이 데스크톱 한 줄 `2:1:1` 비율과 같은 높이로 정렬된다.
- 홈에서 예약 링크는 정확히 두 개이고 중간 예약 띠와 최종 사진이 존재하지 않는다.
- 최종 CTA는 데스크톱 두 열, 모바일 한 열이며 사진 없이 Navy 배경으로 마감된다.
- 블로그 데이터, 공개 경로, 전화, 메일, SNS, 예약 URL이 유지된다.
- `npm run typecheck`, `npm test`, `npm run build`가 모두 통과한다.

## 승인된 결론

리틀브라스 홈은 순수 흰색을 기본 면으로 사용하고 Navy를 구분과 행동에 집중한다. 실제 사진은 악기·교육·공간을 설명하는 증거로만 남기며, 교육 기준에는 승인된 `brand-wall-01.webp` 세로 보정본을 사용한다. 예약 행동은 첫 화면과 마지막 화면의 두 번으로 제한하며, 마지막 영역은 사진 없는 6:4 텍스트 CTA로 마감한다.
