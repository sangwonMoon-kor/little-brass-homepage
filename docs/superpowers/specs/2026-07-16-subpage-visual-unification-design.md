# 리틀브라스 서브 페이지 시각 통합 명세

## 상태와 범위

이 문서는 `2026-07-16-homepage-visual-refinement-design.md`에서 승인·구현된 화이트·Navy·Brass 홈 디자인을 커리큘럼, 교육철학, 갤러리, 찾아오시는 길 페이지에 확장하는 기준을 정의한다. 홈의 섹션 순서와 예약 링크 두 개 계약은 변경하지 않는다.

## 목표

- 서브 페이지의 따뜻한 베이지 대면을 제거하고 홈과 같은 순수 흰색 기본 면을 사용한다.
- Navy는 페이지 구분과 핵심 정보, Brass는 번호와 작은 레이블에만 사용한다.
- 네 페이지가 동일한 템플릿을 반복하지 않고 각 방문 목적에 맞는 첫 화면과 본문 구조를 갖게 한다.
- 커리큘럼의 악기 소개와 악기 선택 탭 중복을 하나의 선택 구조로 합친다.
- 찾아오시는 길을 사진 소개 페이지가 아니라 주소·지도·연락·교통을 빠르게 해결하는 페이지로 만든다.
- 모바일에서 제목 줄바꿈, 섹션 간격, 가로 스크롤과 과도한 세로 길이를 정리한다.
- Emil Kowalski의 디자인 엔지니어링 원칙에 따라 목적 없는 모션을 추가하지 않고, 반응 속도·상태 피드백·접근성 같은 보이지 않는 완성도를 높인다.

## 비목표

- 홈 화면의 승인된 구조, 이미지와 예약 링크 개수를 바꾸지 않는다.
- 전화, 이메일, 주소, 운영시간, 예약 URL, SNS URL과 블로그 데이터를 바꾸지 않는다.
- 새 UI 프레임워크, 모션 라이브러리, 지도 SDK 또는 런타임 의존성을 추가하지 않는다.
- 실제 학원 사진을 AI로 합성하거나 존재하지 않는 공간·인물·경력을 만들지 않는다.
- `littlebrass.com` DNS 연결 작업은 이 디자인 구현과 별도로 계속 진행한다.

## 검토한 접근과 선택

1. **색상만 통일**: 빠르지만 공통 인트로 반복, 커리큘럼 중복과 오시는 길의 낮은 실용성이 그대로 남아 제외한다.
2. **공통 시각 체계 + 페이지별 구조 개선**: 홈과 연결되는 브랜드 일관성을 만들면서 기존 콘텐츠와 Hono 구조를 보존한다. 이번 작업의 선택안이다.
3. **전체 페이지 전면 재작성**: 차별화 폭은 크지만 콘텐츠 검증과 회귀 위험이 커 현재 범위를 넘으므로 제외한다.

## 공통 디자인 시스템

### 색상

- Base White `#FFFFFF`: 모든 페이지의 기본 배경
- Navy `#102B4E`: 페이지 구분 띠, 선택 상태, 핵심 버튼
- Deep Navy `#0A1C33`: 푸터와 제한적인 강한 구분
- Brass `#B79035`: 번호, 작은 레이블, 상태 포인트
- Ink `#17202C`: 큰 제목과 본문
- Muted `#626A73`: 보조 설명
- Rule `#DEDEDE`: 섹션과 목록의 1px 구분선
- Soft `#F8F8F6`: 지도 보조 패널처럼 필요한 제한적 보조 면

그라디언트, 큰 그림자, 둥근 카드 묶음과 넓은 금색 배경은 사용하지 않는다. 사진은 평평한 사각형으로 유지하고 구분은 여백과 얇은 선으로 만든다.

### 타이포그래피와 밀도

- 브랜드 워드마크는 `Cormorant Garamond`, 한국어 제목은 `Noto Serif KR`, UI와 본문은 `Noto Sans KR`을 유지한다.
- 데스크톱 페이지 제목은 `52~72px`, 섹션 제목은 `34~44px`, 본문은 `15~17px` 범위로 사용한다.
- 모바일 페이지 제목은 `42~52px`를 상한으로 하고 `word-break: keep-all`과 페이지별 최대 폭을 사용한다.
- 공통 첫 화면의 하단 빈 공간을 줄이고 다음 섹션의 시작이 첫 스크롤 안에서 일부 보이게 한다.

## 공통 셸과 첫 화면

- 서브 페이지의 내비게이션은 흰 배경, Navy 텍스트, Rule 하단선으로 홈의 스크롤 상태와 같은 인상을 유지한다.
- 기존 `PageIntro`는 `variant`를 받아 페이지별 레이아웃을 표현한다. 기본 구조는 번호·제목·설명·선택적 이미지로 유지하되 모든 페이지가 같은 작은 우측 사진을 반복하지 않는다.
- 커리큘럼과 교육철학은 텍스트와 사진의 두 열 구조를 사용한다.
- 갤러리는 인트로 사진을 제거하고 제목 다음에 실제 사진 그리드가 바로 시작되게 한다.
- 찾아오시는 길은 인트로 사진을 제거하고 주소·운영시간·지도 행동을 첫 화면에 배치한다.
- 서브 페이지 상단의 `원데이 클래스` 링크는 일반 내비게이션 링크와 같은 위계로 낮춘다. 강조 예약 버튼은 찾아오시는 길의 방문 안내와 각 페이지 마지막 행동에서만 사용한다.

## 페이지별 구조

### 커리큘럼

1. 화이트 인트로: 제목, 짧은 과정 설명, 승인된 `brand-wall-01.webp` 사진
2. 실기 과정 소개
3. 악기 선택 탭: 트럼펫·호른·트롬본·유포늄 이름과 음색 설명을 한 컨트롤 안에 결합
4. 선택 악기의 입문·초급·중급·고급 단계
5. 군악대·예고·음대 목표 안내
6. 음악이론반
7. 수업 안내와 상담 행동

기존 `instrument-guide` 네 칸은 제거한다. 탭은 데스크톱에서 4열 선택 띠, 모바일에서 가로 스크롤 없이 2×2 격자로 표시한다. 선택 전환은 콘텐츠가 즉시 바뀌고 색상·밑줄만 `180~220ms`로 전환한다. 키보드 방향키, Home, End 동작은 유지한다.

### 교육철학

1. 화이트 인트로: 제목과 학원 대기 공간 사진
2. 원장 소개: 복도 사진과 두 단락의 소개
3. Navy 선언 영역: `작은 금관악기 연주자, 리틀브라스`와 이름의 의미
4. 세 가지 수업 원칙
5. 금관악기 전문·입시·온오프라인·연주 경험 목록

기존의 베이지 `name-story` 면은 Navy로 바꾸고 흰 제목·반투명 흰 본문·Brass 레이블을 사용한다. 장식적인 프로필 카드는 추가하지 않으며 실제로 제공된 정보만 사용한다.

### 갤러리

1. 화이트 텍스트 인트로
2. 실제 학원 공간 사진 그리드
3. 수업과 연주회 기록을 블로그로 연결하는 얇은 Navy 마감 띠

인트로의 복도 사진을 제거해 사진 설명이 두 번 시작되는 느낌을 없앤다. 대표 대기 공간 사진은 데스크톱에서 넓은 두 열을 차지하고 나머지는 1열 단위로 배치한다. 모바일은 한 열이며 모든 사진 높이를 강제로 같게 만들지 않고 가로 사진과 세로 사진의 성격을 유지한다. 호버 확대는 fine pointer에서만 `scale(1.015)`와 `220ms` 이하로 제한한다.

### 찾아오시는 길

1. 화이트 인트로: 제목, 역에서의 거리, 주소, 지도 버튼
2. 방문 정보: 전화·이메일·평일·토요일·일요일
3. Navy 교통 안내: 지하철·버스·주차 세 항목
4. 노란 출입문 사진과 `5층 501호` 도착 안내

외부 지도 SDK는 추가하지 않는다. 네이버 지도 링크를 가장 강한 행동으로 유지하고 주소는 선택·복사 가능한 일반 텍스트로 제공한다. 모바일 제목은 `찾아오시는 길`이 `찾아오 / 시는 길`처럼 분절되지 않도록 전용 크기와 최대 폭을 사용한다.

## 모션과 상호작용

- 버튼 눌림은 현재의 `scale(0.97)`과 `140ms` ease-out을 유지한다.
- 탭, 링크, 메뉴의 상태 변화는 `transform`, `opacity`, `color`, `background-color`만 사용하고 `transition: all`을 금지한다.
- 스크롤 진입은 현재 `18px` 이동을 `10px`로 낮추고 `220~260ms`의 강한 ease-out을 사용한다.
- 키보드로 탭을 바꿀 때는 추가 이동 애니메이션을 재생하지 않는다.
- 모바일 메뉴는 자주 사용하는 기능이므로 느린 드로어 애니메이션을 추가하지 않고 즉시 열리며 아이콘만 `180~220ms`로 전환한다.
- `prefers-reduced-motion`에서는 이동을 제거하고 색상·불투명도 변화만 최소 시간으로 유지한다.

## 접근성과 반응형

- 모든 페이지는 `h1`을 하나만 사용하고 기존 landmark 구조를 유지한다.
- 모든 인터랙티브 요소는 최소 44px 터치 영역과 명확한 포커스 링을 갖는다.
- 이미지의 구체적인 대체 텍스트, `width`·`height`, 하단 이미지의 lazy loading을 유지한다.
- `1440px`, `768px`, `390px`에서 가로 넘침이 없어야 한다.
- 모바일의 페이지 제목과 섹션 제목은 `word-break: keep-all`; 긴 주소와 이메일은 `overflow-wrap: anywhere`로 처리한다.
- 커리큘럼 탭은 선택 상태를 색상뿐 아니라 `aria-selected`와 구조적 표시로 전달한다.

## 파일 경계

- `src/components/PageIntro.tsx`: 페이지별 인트로 variant와 선택적 행동 영역
- `src/pages/CurriculumPage.tsx`: 악기 가이드 제거, 설명이 결합된 탭과 단계 구성
- `src/pages/PhilosophyPage.tsx`: Navy 이름 선언 영역과 섹션 위계
- `src/pages/GalleryPage.tsx`: 사진 없는 인트로와 바로 이어지는 그리드
- `src/pages/LocationPage.tsx`: 주소·지도 중심 인트로, 교통과 출입문 안내
- `src/components/Navigation.tsx`: 서브 페이지 예약 링크의 시각 위계 조정에 필요한 클래스
- `public/static/style.css`: 공통 토큰 재매핑, 페이지 variant, 반응형과 모션
- `tests/accessibility-structure.test.ts`: 페이지별 단일 h1, 탭·링크·랜드마크 계약
- `tests/assets.test.ts`: 이미지 크기 힌트·lazy loading·페이지별 승인 자산 계약
- `tests/build-output.test.ts`: 화이트·Navy 토큰, 금지된 모션과 반응형 회귀 계약

새 상태 관리, 새 런타임 의존성 또는 페이지별 별도 JavaScript 파일은 추가하지 않는다.

## 검증 기준

- 네 서브 페이지의 기본 배경이 `#FFFFFF`이고 큰 베이지 면이 남지 않는다.
- 각 페이지 첫 화면이 같은 작은 우측 사진 템플릿을 반복하지 않는다.
- 커리큘럼에서 악기 네 종 설명은 선택 탭에만 한 번 나타난다.
- 갤러리는 인트로 사진 없이 첫 사진 그리드가 빠르게 시작된다.
- 찾아오시는 길의 첫 화면에서 주소와 네이버 지도 행동을 확인할 수 있다.
- 모바일 `390px`에서 `찾아오시는 길` 제목이 어색한 음절 단위로 분리되지 않는다.
- 내비게이션, 모바일 메뉴, 커리큘럼 키보드 탭, 외부 링크와 404 동작이 유지된다.
- `npm run typecheck`, `npm test`, `npm run build`가 모두 통과한다.

## 승인된 결론

서브 페이지는 홈의 화이트·Navy·Brass 시각 체계를 공유하되, 커리큘럼은 선택과 단계, 교육철학은 선언과 원칙, 갤러리는 사진, 찾아오시는 길은 방문 행동을 중심으로 서로 다른 첫 화면과 정보 흐름을 갖는다. 모션은 상태 이해와 눌림 피드백에만 사용하고 장식보다 밀도·위계·반응성을 우선한다.
