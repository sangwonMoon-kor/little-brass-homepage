# 리틀브라스 사진·커리큘럼·공동원장·갤러리 마감 명세

## 상태와 범위

이 문서는 홈과 서브 페이지에 이미 적용된 White·Navy·Brass 시각 체계를 유지하면서, 실제 학원 사진의 용도와 페이지별 정보 밀도를 다시 정리하는 기준을 정의한다. 이번 범위는 홈의 악기 카드와 `WHY LITTLE BRASS`, 커리큘럼 전체, 교육철학의 공동원장 소개, 갤러리의 공간·수업 분리에 한정한다.

## 목표

- 홈의 네 악기 사진을 같은 촬영 문법과 비율로 정리해 상품 목록처럼 단정하게 보이게 한다.
- `WHY LITTLE BRASS`의 한 장짜리 사진 영역을 실제 수업과 강사진을 함께 보여주는 두 장 구성으로 바꿔 빈 공간을 줄인다.
- 커리큘럼 첫 화면과 음악이론반에서 맥락에 맞지 않는 인물 사진을 제거하고, 텍스트와 구분선 중심의 읽기 쉬운 흐름으로 전환한다.
- 커리큘럼의 제목, 단계 설명, 수업 안내가 현재보다 분명한 크기 차이를 갖도록 타이포그래피와 행간을 조정한다.
- 교육철학에서 김효민 원장과 안세은 원장을 동등한 공동원장으로 소개한다.
- 갤러리를 `학원 공간`과 `수업과 무대` 두 기록으로 나눠 실제 학원 환경도 충분히 보여준다.
- Emil Kowalski의 디자인 엔지니어링 원칙에 따라 장식보다 정보 위계, 즉각적인 반응, 일관된 여백과 접근성을 우선한다.

## 비목표

- 홈 히어로 영상, 예약 링크 수, 내비게이션 구조, 주소·전화·이메일·SNS·예약 URL은 바꾸지 않는다.
- 존재하지 않는 강사 경력, 학력, 담당 악기나 개인 소개를 작성하지 않는다.
- 실제 학원이나 원장 사진을 AI로 합성하지 않는다.
- 악기 형태를 생성형 이미지로 새로 만들지 않는다. 악기 사진은 현재 승인된 실물 이미지의 선택·크롭·색조·배경 정돈으로 통일한다.
- 새 UI 프레임워크, 애니메이션 라이브러리 또는 런타임 의존성을 추가하지 않는다.
- 찾아오시는 길 페이지와 Cloudflare·DNS 설정은 이번 범위에서 변경하지 않는다.

## 검토한 접근과 선택

### 악기 카드

1. **깨끗한 카탈로그 구성 — 선택안**: 네 장을 동일한 밝은 배경, 동일한 4:5 프레임과 피사체 크기로 정돈한다. 실제 악기의 형태를 보존하면서 현재보다 차분하고 전문적으로 보인다.
2. **학원 진열장 사진 크롭**: 실제성은 높지만 네 악기를 같은 선명도와 각도로 확보하기 어렵고, 유리 반사와 주변 악기가 카드마다 다르게 남는다.
3. **사진 제거 후 타이포그래피만 사용**: 가장 단정하지만 사용자가 요청한 악기 사진 개선을 충족하지 못한다.

선택안은 기존 실물 악기 원본을 사용한다. 큰 소품과 손이 보이는 트럼펫 사진은 피사체 중심 크롭과 밝은 여백으로 손의 비중을 줄이고, 네 장 모두 배경 밝기·온도와 악기 배율을 맞춘다. 배경은 흰색에 가까운 Warm White로 통일하고 금속의 실제 색과 디테일은 과도하게 보정하지 않는다.

### 홈 `WHY LITTLE BRASS`

1. **두 장의 비대칭 세로 콜라주 — 선택안**: 수업 사진과 공동원장 사진을 위아래로 엇갈리게 배치해 실제 수업과 운영 주체를 한 번에 보여준다.
2. **한 장을 더 크게 확대**: 구현은 단순하지만 현재의 빈 왼쪽 영역과 단조로운 리듬이 남는다.
3. **가로 슬라이더**: 사진은 많이 보여줄 수 있지만 자동 전환과 조작 요소가 내용보다 앞서고 불필요한 모션이 생긴다.

### 커리큘럼

1. **사진 없는 텍스트 중심 편집형 흐름 — 선택안**: 첫 화면과 음악이론반의 불필요한 인물 사진을 제거하고, 제목·요약·구분선·정보 표로 내용을 빠르게 읽게 한다.
2. **다른 강사 사진으로 교체**: 인물 사진과 해당 과정의 직접적인 연결을 증명하기 어려워 같은 문제가 반복된다.
3. **악기 또는 악보 소품 사진 사용**: 맥락은 맞지만 지금 필요한 타이포그래피와 간격 문제를 해결하지 못한다.

## 공통 시각 기준

- Base White `#FFFFFF`: 모든 기본 면
- Navy `#102B4E`: 제목, 선택 상태, 핵심 행동
- Brass `#B79035`: 번호, 작은 레이블과 제한적인 포인트
- Ink `#17202C`: 본문 핵심 텍스트
- Muted `#626A73`: 보조 설명
- Rule `#DEDEDE`: 모든 섹션과 목록의 1px 구분선
- Warm White `#F7F6F2`: 악기 이미지 프레임에만 제한적으로 사용

그라디언트, 큰 그림자, 둥근 카드 묶음과 넓은 색 면은 추가하지 않는다. 사진은 직선적인 사각형으로 유지하고, 섹션 구분은 여백과 1px Rule로 만든다.

## 페이지별 상세 설계

### 홈: 네 가지 금관악기

- 네 카드의 이미지 영역은 데스크톱에서 동일한 `4:5` 비율을 사용한다.
- 사진은 Warm White 배경 위에서 악기의 중심과 외곽 여백을 통일한다. 피사체는 프레임을 가득 채우지 않고 상하좌우에 숨 쉴 여백을 둔다.
- 트럼펫·호른·트롬본·유포늄은 각각 한 악기만 시선의 중심이 되게 크롭한다. 주변 소품, 손과 가구가 제목보다 강하게 보이지 않아야 한다.
- 캡션은 이미지 아래의 얇은 Navy 영역 또는 흰 배경의 선형 정보 영역 중 하나로 통일하며, 높이를 현재보다 낮춘다.
- 한글 악기명은 가장 크게, 음색 설명은 보조, 영문명은 Brass의 작은 레이블로 유지한다.
- 카드 전체에 그림자나 둥근 모서리를 추가하지 않는다.

### 홈: `WHY LITTLE BRASS`

- 왼쪽 `education-photo` 한 장을 `education-photo-stack` 두 장으로 변경한다.
- 위/주 이미지: `/static/images/academy/ensemble-lesson-01.webp` — 실제 합주 수업.
- 아래/보조 이미지: `/static/images/academy/faculty-duo-brass-01.webp` — 공동원장과 금관악기.
- 데스크톱은 큰 수업 사진과 폭이 조금 좁은 공동원장 사진이 세로로 겹치지 않고 엇갈려 이어지는 비대칭 편집 구성을 사용한다.
- 사진 사이에는 `16~24px` 간격과 한쪽에 1px Navy 기준선을 사용한다. 장식용 배경 블록이나 그림자는 사용하지 않는다.
- 오른쪽 제목·설명·세 항목의 현재 순서는 유지한다. 두 열의 전체 높이가 비슷해지도록 사진 비율과 목록 간격을 조정한다.
- 모바일에서는 두 사진을 한 열로 쌓되, 보조 사진 높이를 더 낮게 유지하고 그 뒤 텍스트가 시작되게 한다.

### 커리큘럼: 인트로와 섹션 흐름

- `PageIntro`의 커리큘럼 사진을 제거한다.
- 커리큘럼 인트로는 번호·제목·설명을 한 개의 넓은 텍스트 그리드로 구성한다. 제목과 설명을 한 줄에 억지로 맞추지 않고, 설명 폭을 제한해 읽기 길이를 유지한다.
- 인트로 하단과 `실기 과정` 사이에는 1px Rule을 사용하고, 데스크톱의 총 공백은 약 `72~88px`, 모바일은 약 `48~56px` 범위로 줄인다.
- `실기 과정`, 탭, 단계 목록, 목표 안내 사이의 구분은 넓은 빈 면 대신 Rule과 반복되는 내부 패딩으로 만든다.
- 탭 상호작용과 키보드 접근성 계약은 그대로 유지한다.

### 커리큘럼: 타이포그래피

- 데스크톱 섹션 제목: `44~52px`, 줄높이 `1.18~1.25`.
- 모바일 섹션 제목: `32~36px`, 줄높이 `1.25`.
- 악기 탭 이름과 단계 제목: `20~22px` 이상.
- 단계 요약과 수업 설명: `15~16px`, 줄높이 `1.65~1.75`.
- `dt`, 기간, 번호와 보조 레이블: `13~14px`; 명도 대비는 현재보다 높인다.
- `수업 안내`의 각 항목 제목은 `21~24px`, 부제와 사실 정보는 `14~16px`, 설명은 `15~16px`로 올린다.
- 긴 설명은 한 줄에 과도하게 넓어지지 않도록 본문 최대 폭을 둔다.

### 커리큘럼: 뮤토랑 음악이론반

- `/static/images/academy/faculty-duo-presentation-01.webp`와 캡션을 제거한다.
- 섹션은 사진 없는 텍스트형 2열 ledger로 변경한다.
- 왼쪽은 kicker·제목·도입 설명, 오른쪽은 대상·시간·방식·수강 정보와 네 가지 학습 주제를 배치한다.
- 데스크톱 두 열 사이에는 넓은 공백 대신 1px 세로 Rule 또는 명확한 열 간격을 사용한다.
- 모바일에서는 제목, 수업 정보, 학습 주제 순으로 한 열에 쌓고 항목 사이 Rule을 유지한다.

### 교육철학: 공동원장

- 페이지 인트로의 `/static/images/academy/faculty-duo-standing-01.webp`는 두 공동원장을 함께 보여주는 이미지로 유지한다.

  - 표시명 `김효민 원장`
  - 레이블 `공동원장`
- 안세은 공동원장:

- 기존 이름 이야기, 수업 원칙, 전문성 섹션은 공동원장 소개 뒤에 현재 순서로 이어진다.

## 모션과 상호작용

- 새 자동 재생, 캐러셀, 스크롤 고정 또는 장식 모션을 추가하지 않는다.
- 사진 호버는 fine pointer에서만 `scale(1.01~1.015)`, `220ms` 이하로 제한한다.
- 탭은 `color`, `background-color`, `border-color`만 짧게 전환하며 `transition: all`을 사용하지 않는다.
- `prefers-reduced-motion`에서는 이미지 확대와 이동을 제거한다.

## 접근성과 반응형

- 모든 페이지는 `h1`을 하나만 유지하고 섹션 제목 연결에 `aria-labelledby`를 사용한다.
- 인물 사진의 대체 텍스트는 이름과 역할을 정확히 포함하고, 외모에 대한 불필요한 설명을 넣지 않는다.
- 사진은 실제 출력 비율에 맞는 `width`·`height`, 하단 자산의 `loading="lazy"`와 `decoding="async"`를 유지한다.
- `1440px`, `768px`, `390px`에서 가로 넘침이 없어야 한다.
- 모바일의 제목은 `word-break: keep-all`을 사용하고, 커리큘럼 표 정보는 잘리거나 가로 스크롤되지 않게 한 열 또는 2열로 재배치한다.
- 모든 탭과 링크는 최소 44px 터치 영역, 명확한 포커스 표시를 유지한다.

- `tests/assets.test.ts`: 승인 자산, 인물 매핑, 이미지 크기·lazy loading 계약
- `tests/accessibility-structure.test.ts`: 커리큘럼과 갤러리 섹션, 공동원장 이름과 탭 접근성 계약
- `tests/build-output.test.ts`: 타이포그래피·반응형·금지 모션 회귀 계약

새 상태 관리, 새 런타임 의존성 또는 페이지별 별도 JavaScript 파일은 추가하지 않는다.

## 검증 기준

- 홈의 네 악기 카드가 같은 밝은 배경, 4:5 비율, 피사체 배율과 캡션 높이를 공유한다.
- `WHY LITTLE BRASS` 왼쪽에 실제 합주 수업과 공동원장 사진이 두 장 보이고, 데스크톱과 모바일 모두 빈 면이 과도하지 않다.
- 커리큘럼 인트로와 음악이론반에 인물 사진이 남지 않는다.
- 커리큘럼의 실기 단계와 `수업 안내`가 390px에서도 확대 없이 읽을 수 있는 크기와 행간을 갖는다.
- 교육철학에 `김효민 원장`, `안세은 원장`, `공동원장`이 동등한 위계로 표시된다.
- 김효민은 `instructor-portrait-01.webp`, 안세은은 `instructor-trumpet-portrait-01.webp`와 정확히 연결된다.
- 갤러리에 `학원 공간`과 `수업과 무대`가 각각 독립된 제목과 실제 사진 묶음으로 표시된다.
- 페이지 전체는 White base, Navy 제목, Brass 소형 포인트, 1px Rule의 시각 체계를 유지한다.
- 키보드 커리큘럼 탭, 모바일 메뉴, 예약 링크, 외부 링크와 404 동작이 유지된다.
- `npm run typecheck`, `npm test`, `npm run build`, `git diff --check`가 모두 통과한다.

## 승인된 결론

이번 마감은 사진 수를 단순히 늘리는 작업이 아니다. 홈에서는 악기 사진을 한 문법으로 정돈하고 `WHY LITTLE BRASS`를 실제 수업과 공동원장 이야기로 채운다. 커리큘럼은 관련 없는 사진을 빼고 제목·설명·구분선의 밀도를 높인다. 교육철학은 김효민·안세은 두 공동원장을 동등하게 소개하며, 갤러리는 학원 공간과 수업·무대 기록을 분리한다. 모든 변화는 기존 White·Navy·Brass 체계와 실제 제공된 정보 안에서만 이뤄진다.
